# Markdown 语法使用时机指南

## 🎯 标题的使用时机

### 使用 `# 一级标题`
- **场景**：文档主标题、文章标题
- **示例**：`# 安装指南`
- **注意**：一个文档只用一次

### 使用 `## 二级标题`
- **场景**：主要章节划分
- **示例**：`## 安装步骤`、`## 配置说明`
- **注意**：用于逻辑上的大区块

### 使用 `### 三级标题` 及以下
- **场景**：小节细分、功能点
- **示例**：`### 数据库配置`、`#### 连接参数`
- **规则**：不要跳级（如从二级直接到四级）

---

## 📝 正文段落的使用

### 使用普通段落
- **场景**：说明性文字、描述性内容
- **格式**：直接写文字，段落间空一行
- **示例**：
```markdown
这是第一个段落，包含连续的句子。

这是第二个段落，与前一段有空行分隔。
```

### 使用 **粗体**
- **场景**：强调关键术语、重要概念
- **示例**：`**务必**备份数据后再操作`
- **避免**：大段文字加粗

### 使用 *斜体*
- **场景**：术语引用、轻微强调、外来语
- **示例**：`这种方法称为*递归调用*`
- **注意**：中文文档中少用

---

## 📋 列表的使用时机

### 使用无序列表 `-`
- **场景**：并列项目、特性列表、步骤不分先后
- **示例**：
```markdown
支持的操作系统：
- Windows 10+
- macOS 12+
- Ubuntu 20.04+
```

### 使用有序列表 `1.`
- **场景**：操作步骤、有顺序的流程
- **示例**：
```markdown
安装步骤：
1. 下载安装包
2. 运行安装程序
3. 重启电脑
```

### 使用任务列表 `- [ ]`
- **场景**：检查清单、待办事项
- **示例**：
```markdown
准备工作：
- [ ] 安装 Node.js
- [ ] 配置环境变量
- [x] 已完成项
```

## 💻 代码相关的使用

### 使用行内代码 `code`
- **场景**：文件名、命令、变量名、简短代码
- **示例**：`使用 npm install 命令`
- **注意**：用于简短的技术术语

### 使用代码块 ` ``` `
- **场景**：多行代码、配置示例、终端输出
- **示例**：
````markdown
```bash
# 安装依赖
npm install
npm start
```
````

### 使用语法高亮
- **场景**：让代码更易读
- **示例**：` ```python `、` ```javascript `
- **规则**：标注正确的语言类型

## 📊 表格的使用时机

### 使用表格
- **场景**：参数说明、功能对比、数据展示
- **示例**：
```markdown
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `host` | string | localhost | 服务器地址 |
| `port` | number | 8080 | 监听端口 |
```

### 表格适用情况
✅ **适合**：配置参数、API接口、版本对比  
❌ **不适合**：复杂数据结构、大量文本内容

---

## ❗ 引用块的使用

### 使用 `> 引用`
- **场景**：说明、注释、注意事项、引用来源
- **示例**：
```markdown
> **注意**：此操作不可逆，请谨慎执行
> 
> 来自官方文档第 3.2 节
```

### 嵌套引用
- **场景**：多层说明、对话引用
- **示例**：
```markdown
> 主要说明
>> 补充说明
>>> 详细解释
```

## 🎨 特殊元素的使用

### 使用分割线 `---`
- **场景**：章节分隔、场景转换
- **示例**：在主要部分之间使用
- **注意**：不要过度使用

### 使用脚注 ``
- **场景**：补充说明、引用来源
- **示例**：
```markdown

: 2004 年发布
```

## ⚠️ 常见使用误区

### 应该避免的情况
| 错误用法 | 问题 | 正确做法 |
|----------|------|----------|
| 太多 `####` 小标题 | 结构过于碎片化 | 合并相关内容 |
| 大段文字无分段 | 阅读疲劳 | 每3-5句分一段 |
| 表格塞满长文本 | 难以阅读 | 改用列表或段落 |
| 过度使用粗体 | 失去重点 | 关键处才强调 |

### 排版最佳实践
1. **标题序列化**：保持 `# → ## → ###` 的层级
2. **段落长度**：3-5句话为一段，保持可读性
3. **列表分组**：相关项目放在同一列表中
4. **代码注释**：复杂代码添加说明注释
