## 第1讲：C++ 语言简介与开发环境搭建

### 概念

C++ 是一种通用的、静态类型的、编译式编程语言，由 Bjarne Stroustrup 于 1979 年在贝尔实验室开发，最初被称为 "C with Classes"（带类的 C）。它是在 C 语言的基础上增加了面向对象编程、泛型编程和模板等特性而形成的。C++ 既保留了 C 语言高效、灵活、接近硬件的特点，又引入了高级语言的抽象能力，因此被广泛应用于系统软件、游戏开发、嵌入式系统、高性能计算等领域。C++ 经历了多次标准更新，目前主流标准包括 C++11、C++14、C++17 和 C++20，每一次更新都为语言注入了新的活力。

### 原理

C++ 是一种**编译型语言**，源代码（`.cpp` 文件）需要经过预处理、编译、汇编、链接四个阶段，最终生成可执行的机器码。与 Python 等解释型语言不同，C++ 程序在运行前已经完全转换为机器指令，因此执行效率极高。C++ 的设计哲学是"零开销抽象"（zero-overhead abstraction），即你为高级特性付出的运行时代价几乎为零——你不使用的特性就不会带来额外开销。这一理念使得 C++ 既能提供强大的抽象能力，又能保持与 C 语言媲美的性能。此外，C++ 由国际标准化组织（ISO）维护，保证了跨平台的一致性，同一份代码可以在 Windows、Linux、macOS 上编译运行。

### 例子

下面是 C++ 最经典的入门程序——"Hello, World!"：

```cpp
#include <iostream>  // 引入输入输出流头文件

int main() {
    std::cout << "Hello, World!" << std::endl;  // 输出文本到控制台
    return 0;  // 返回 0 表示程序正常结束
}
```

**编译与运行**（以 g++ 为例）：

```bash
g++ -std=c++17 -o hello hello.cpp   # 编译
./hello                              # 运行
```

**程序解析**：
- `#include <iostream>` 是预处理指令，引入标准输入输出库
- `int main()` 是程序入口，每个 C++ 程序都必须有一个 `main` 函数
- `std::cout` 是标准输出流对象，`<<` 是插入运算符
- `std::endl` 表示换行并刷新缓冲区
- `return 0;` 向操作系统返回退出状态码

**常用开发工具**：
- 编译器：GCC（g++）、Clang、MSVC
- IDE：Visual Studio、CLion、VS Code + 插件
- 构建工具：CMake、Make

### 总结

本讲介绍了 C++ 的历史定位与编译运行机制。关键要点如下：第一，C++ 是编译型语言，源码需经过编译链接才能执行，这决定了它的高性能特性；第二，C++ 的核心设计哲学是"零开销抽象"，既高效又强大；第三，编写 C++ 程序的基本框架是 `#include` 头文件 + `main` 函数。常见注意事项：初学者应选择一款顺手的 IDE 以降低环境配置成本；编译时建议指定较新的标准（如 `-std=c++17`）以使用现代特性；务必注意 `main` 函数必须返回 `int` 类型。掌握这些基础后，你就可以开始编写真正的 C++ 程序了。

## 第2讲：基本数据类型与变量

### 概念

数据类型是编程语言对数据的分类方式，它决定了变量占用多少内存、能存储什么范围的值、以及支持哪些运算。C++ 提供了丰富的**内置基本数据类型**，主要分为四大类：布尔型（`bool`）、字符型（`char`）、整型（`int`、`short`、`long`、`long long`）和浮点型（`float`、`double`、`long double`）。此外，C++11 引入了 `auto` 类型推导和 `nullptr` 空指针等现代特性，让类型系统更加灵活。变量则是程序中用来存储数据的命名内存单元，每个变量都有明确的类型，在声明时确定，此后不可改变。

### 原理

C++ 是**静态类型语言**，编译器在编译期就必须知道每个变量的类型，从而为其分配正确大小的内存并生成高效的机器码。不同类型占用的字节数由编译器和平台决定，但标准规定了最小范围。例如，`int` 通常占 4 字节（32 位），可表示约 ±21 亿的整数；`double` 占 8 字节，提供约 15-16 位有效数字的精度。C++ 还通过 `signed`/`unsigned` 修饰符区分有符号和无符号类型——有符号类型可表示正负数，无符号类型只能表示非负数但范围翻倍。变量在声明时会在栈上分配内存（局部变量）或在堆上分配（通过 `new`），并可通过初始化赋予初值。未初始化的局部变量包含随机值（未定义行为），这是初学者最常踩的坑之一。

### 例子

```cpp
#include <iostream>
#include <limits>

int main() {
    // 基本数据类型声明与初始化
    bool isReady = true;              // 布尔型，占 1 字节
    char grade = 'A';                 // 字符型，占 1 字节
    int age = 25;                     // 整型，通常 4 字节
    long long bigNum = 9223372036854775807LL;  // 长整型，8 字节
    float pi = 3.14f;                 // 单精度浮点，4 字节（注意 f 后缀）
    double e = 2.718281828459045;     // 双精度浮点，8 字节
    unsigned int count = 100u;        // 无符号整型

    // 使用 auto 自动推导类型（C++11）
    auto score = 95.5;                // 推导为 double
    auto name = "Alice";              // 推导为 const char*

    // 输出各类型的大小和值
    std::cout << "bool 大小: " << sizeof(bool) << " 字节" << std::endl;
    std::cout << "int 大小: " << sizeof(int) << " 字节" << std::endl;
    std::cout << "double 大小: " << sizeof(double) << " 字节" << std::endl;
    std::cout << "age = " << age << ", pi = " << pi << std::endl;

    // int 的取值范围
    std::cout << "int 范围: " << std::numeric_limits<int>::min()
              << " ~ " << std::numeric_limits<int>::max() << std::endl;

    // 常量定义
    const double PI = 3.141592653589793;  // 运行时常量
    constexpr int MAX_SIZE = 100;          // 编译期常量（C++11）

    return 0;
}
```

**输出示例**：
```
bool 大小: 1 字节
int 大小: 4 字节
double 大小: 8 字节
age = 25, pi = 3.14
int 范围: -2147483648 ~ 2147483647
```

### 总结

本讲讲解了 C++ 的基本数据类型与变量机制。关键要点：第一，C++ 是静态类型语言，变量类型在编译期确定，这保证了运行效率但要求开发者对类型有清晰认识；第二，整型有 `short`/`int`/`long`/`long long` 四档，浮点型有 `float`/`double` 两档，日常编程推荐默认使用 `int` 和 `double`；第三，`auto` 关键字可以简化类型声明，但不应滥用以免降低代码可读性。常见注意事项：局部变量务必初始化，否则其值是未定义的；浮点数字面量默认是 `double` 类型，赋给 `float` 变量时需加 `f` 后缀；使用 `unsigned` 类型时要小心与有符号数混合运算导致的意外结果；优先使用 `constexpr` 定义编译期常量，它比 `const` 更严格、更高效。

## 第3讲：运算符与表达式

### 概念

运算符是 C++ 中用于执行各种运算的符号，表达式则是由运算符和操作数组成的、能够计算出一个值的代码片段。C++ 的运算符种类繁多，按功能可分为：算术运算符（`+`、`-`、`*`、`/`、`%`）、关系运算符（`==`、`!=`、`<`、`>`、`<=`、`>=`）、逻辑运算符（`&&`、`||`、`!`）、位运算符（`&`、`|`、`^`、`~`、`<<`、`>>`）、赋值运算符（`=`、`+=`、`-=` 等）、自增自减运算符（`++`、`--`）以及条件运算符（`?:`）等。此外，C++ 还支持运算符重载，允许为自定义类型定义运算符行为，这是后续面向对象章节的内容。

### 原理

C++ 运算符的核心机制是**优先级**和**结合性**。优先级决定了多个运算符共存时的计算顺序，例如 `*` 优先于 `+`；结合性则决定了同优先级运算符的计算方向（多数为左结合，赋值和单目运算符为右结合）。当不同类型的操作数参与运算时，C++ 会进行**隐式类型转换**（也叫类型提升），将低精度类型自动转换为高精度类型以避免数据丢失，例如 `int` 与 `double` 运算时 `int` 会提升为 `double`。逻辑运算符 `&&` 和 `||` 具有**短路求值**特性：当结果已能由左操作数确定时，右操作数不再计算，这一特性常用于安全检查。自增自减运算符有前置（`++i`）和后置（`i++`）之分，前置先增后用，后置先用后增，在性能敏感场景下对迭代器等复杂对象应优先使用前置形式。

### 例子

```cpp
#include <iostream>

int main() {
    // 算术运算
    int a = 17, b = 5;
    std::cout << "a + b = " << a + b << std::endl;   // 22
    std::cout << "a / b = " << a / b << std::endl;   // 3（整数除法，截断小数）
    std::cout << "a % b = " << a % b << std::endl;   // 2（取余）
    std::cout << "a / (double)b = " << a / (double)b << std::endl;  // 3.4

    // 自增自减
    int i = 5;
    std::cout << "i++ = " << i++ << std::endl;  // 输出 5，然后 i 变为 6
    std::cout << "++i = " << ++i << std::endl;  // i 先变为 7，输出 7

    // 关系与逻辑运算（短路求值）
    int x = 10;
    if (x > 0 && x < 100) {
        std::cout << "x 在 0~100 之间" << std::endl;
    }

    // 短路求值的实际应用：避免除零
    int divisor = 0;
    if (divisor != 0 && 100 / divisor > 5) {  // divisor 为 0 时不会执行除法
        std::cout << "条件成立" << std::endl;
    } else {
        std::cout << "安全跳过除法" << std::endl;
    }

    // 位运算
    int m = 0b1100;  // 12
    int n = 0b1010;  // 10
    std::cout << "m & n = " << (m & n) << std::endl;   // 8  (1000)
    std::cout << "m | n = " << (m | n) << std::endl;   // 14 (1110)
    std::cout << "m ^ n = " << (m ^ n) << std::endl;   // 6  (0110)
    std::cout << "m << 2 = " << (m << 2) << std::endl; // 48（左移两位相当于乘 4）

    // 条件运算符（三目运算符）
    int max = (a > b) ? a : b;
    std::cout << "较大值: " << max << std::endl;  // 17

    // 复合赋值
    int sum = 0;
    for (int k = 1; k <= 5; ++k) {
        sum += k;  // 等价于 sum = sum + k
    }
    std::cout << "1+2+3+4+5 = " << sum << std::endl;  // 15

    return 0;
}
```

### 总结

本讲介绍了 C++ 丰富的运算符体系。关键要点：第一，运算符的优先级和结合性决定了表达式的计算顺序，但编写代码时不应过度依赖优先级，复杂表达式应使用括号明确意图；第二，整数除法会截断小数部分，若需浮点结果必须进行类型转换；第三，逻辑运算符的短路求值既能提升性能，也能用于安全检查（如避免空指针解引用、除零等）。常见注意事项：不要混淆 `=`（赋值）与 `==`（相等比较），这是极其常见的 bug 来源；位运算的优先级低于关系运算符，使用时务必加括号；后置 `++` 对复杂对象（如迭代器）会产生额外开销，性能敏感代码应使用前置 `++`；取余运算 `%` 只能用于整型，浮点取余需使用 `fmod` 函数。

## 第4讲：控制流程（条件与循环）

### 概念

控制流程是程序决定"执行什么、执行多少次、何时跳转"的机制，是程序逻辑的骨架。C++ 的控制流程语句分为三大类：**条件分支语句**（`if`、`if-else`、`if-else if-else`、`switch`）、**循环语句**（`while`、`do-while`、`for`、范围 `for`）和**跳转语句**（`break`、`continue`、`return`、`goto`）。通过这些语句的组合，程序能够根据不同的输入和状态执行不同的代码路径，实现复杂的业务逻辑。掌握控制流程是编写任何有意义程序的前提。

### 原理

条件语句的原理是基于布尔表达式（值为 `true` 或 `false`）来决定执行路径。`if` 语句在表达式为真时执行其代码块，`else` 提供假分支，`switch` 则根据整型或枚举值跳转到匹配的 `case` 标签，适合多分支等值判断场景。循环语句的核心是**循环条件判断**和**循环变量更新**：`while` 先判断后执行，`do-while` 先执行后判断（至少执行一次），`for` 将初始化、条件、更新三部分集中在一行，最适合计数循环。C++11 引入的**范围 for 循环**（range-based for）简化了容器遍历，底层等价于使用迭代器的传统 `for`。跳转语句中，`break` 立即跳出当前循环或 `switch`，`continue` 跳过本次循环剩余部分直接进入下一次迭代，`return` 退出当前函数，`goto` 则是无条件跳转——现代编程强烈建议避免使用 `goto`，因为它会破坏结构化编程，导致代码难以维护。

### 例子

```cpp
#include <iostream>
#include <vector>
#include <string>

int main() {
    // 1. if-else 条件判断
    int score = 85;
    if (score >= 90) {
        std::cout << "等级: A" << std::endl;
    } else if (score >= 80) {
        std::cout << "等级: B" << std::endl;
    } else if (score >= 60) {
        std::cout << "等级: C" << std::endl;
    } else {
        std::cout << "等级: D" << std::endl;
    }

    // 2. switch 多分支（适合等值判断）
    int day = 3;
    switch (day) {
        case 1: std::cout << "星期一" << std::endl; break;
        case 2: std::cout << "星期二" << std::endl; break;
        case 3: std::cout << "星期三" << std::endl; break;
        case 4: std::cout << "星期四" << std::endl; break;
        case 5: std::cout << "星期五" << std::endl; break;
        default: std::cout << "周末" << std::endl; break;
    }

    // 3. for 循环：计算 1~10 的和
    int sum = 0;
    for (int i = 1; i <= 10; ++i) {
        sum += i;
    }
    std::cout << "1~10 的和: " << sum << std::endl;  // 55

    // 4. while 循环：计算斐波那契数列前 10 项
    int a = 0, b = 1, count = 0;
    std::cout << "斐波那契数列: ";
    while (count < 10) {
        std::cout << a << " ";
        int next = a + b;
        a = b;
        b = next;
        ++count;
    }
    std::cout << std::endl;

    // 5. do-while 循环：至少执行一次
    int num = 0;
    do {
        std::cout << "do-while 执行, num = " << num << std::endl;
        ++num;
    } while (num < 3);

    // 6. 范围 for 循环（C++11）：遍历容器
    std::vector<int> numbers = {10, 20, 30, 40, 50};
    std::cout << "容器元素: ";
    for (int n : numbers) {       // 值传递
        std::cout << n << " ";
    }
    std::cout << std::endl;

    // 使用引用避免拷贝，提高效率
    for (const int& n : numbers) {
        std::cout << n << " ";
    }
    std::cout << std::endl;

    // 7. break 和 continue
    std::cout << "1~20 中的奇数（跳过 15，遇到 19 停止）: ";
    for (int i = 1; i <= 20; ++i) {
        if (i == 15) continue;  // 跳过 15
        if (i == 19) break;     // 遇到 19 终止循环
        if (i % 2 != 0) {
            std::cout << i << " ";
        }
    }
    std::cout << std::endl;  // 输出: 1 3 5 7 9 11 13 17

    return 0;
}
```

### 总结

本讲讲解了 C++ 的控制流程语句。关键要点：第一，`if-else` 适合范围判断，`switch` 适合等值多分支判断，且 `switch` 的 `case` 必须以 `break` 结尾以避免"穿透"（除非有意为之）；第二，`for` 循环最适合已知次数的计数循环，`while` 适合条件驱动的循环，`do-while` 保证至少执行一次；第三，范围 for 循环是遍历容器的现代写法，遍历大对象时应使用引用（`const auto&`）避免拷贝开销。常见注意事项：`switch` 只能判断整型或枚举，不能判断浮点或字符串；循环中务必确保有终止条件，否则会陷入死循环；`continue` 在 `while`/`for` 中行为略有不同——`for` 会先执行更新表达式再判断条件，`while` 直接回到条件判断；尽量避免使用 `goto`，它会破坏代码结构。

# 第2章 函数与复合数据

## 第5讲：函数的定义与使用

### 概念

函数是一段具有特定功能的、可重复调用的代码块，它是 C++ 程序模块化的基本单位。一个函数由**返回类型、函数名、参数列表、函数体**四部分组成。C++ 中的函数参数传递有两种方式：**值传递**（pass by value）将实参的副本传给形参，函数内修改不影响实参；**引用传递**（pass by reference）将实参的别名传给形参，函数内修改直接影响实参。此外，C++ 还支持**默认参数**、**函数重载**（同名不同参）、**内联函数**和**递归**等特性，让函数机制更加灵活强大。

### 原理

函数调用的底层机制涉及**栈帧**（stack frame）。当函数被调用时，系统会在调用栈上为该函数分配一块内存，用于存储局部变量、参数副本和返回地址；函数返回时，这块内存被释放，控制权交还给调用者。值传递之所以不影响实参，正是因为形参是栈帧上的副本。引用传递则不同——引用本质上是变量的别名，编译器通常用指针实现，因此修改引用就是修改原变量。函数重载的实现依赖于**名字修饰**（name mangling），编译器会根据参数类型生成唯一的内部名称，使得同名函数能被正确区分。递归则是函数直接或间接调用自身，每次递归调用都会创建新的栈帧，因此递归深度过大会导致栈溢出。内联函数通过将函数体直接展开到调用处来消除函数调用开销，适合短小频繁调用的函数。

### 例子

```cpp
#include <iostream>
#include <string>

// 1. 基本函数：值传递
int add(int a, int b) {
    return a + b;
}

// 2. 引用传递：交换两个变量的值
void swap(int& x, int& y) {
    int temp = x;
    x = y;
    y = temp;
}

// 3. 默认参数：从右向左设置默认值
double power(double base, int exp = 2) {
    double result = 1.0;
    for (int i = 0; i < exp; ++i) {
        result *= base;
    }
    return result;
}

// 4. 函数重载：同名函数，参数不同
int maxVal(int a, int b) {
    return (a > b) ? a : b;
}
double maxVal(double a, double b) {
    return (a > b) ? a : b;
}
int maxVal(int a, int b, int c) {
    int m = (a > b) ? a : b;
    return (m > c) ? m : c;
}

// 5. 递归：计算阶乘
long long factorial(int n) {
    if (n <= 1) return 1;        // 递归终止条件
    return n * factorial(n - 1); // 递归调用
}

// 6. 递归：斐波那契数列（注意效率问题）
int fib(int n) {
    if (n < 2) return n;
    return fib(n - 1) + fib(n - 2);
}

// 7. const 引用传递：避免拷贝又防止修改（适合大对象）
void printString(const std::string& s) {
    std::cout << s << std::endl;
    // s = "new";  // 编译错误：const 引用不可修改
}

int main() {
    // 值传递
    std::cout << "3 + 5 = " << add(3, 5) << std::endl;

    // 引用传递
    int x = 10, y = 20;
    std::cout << "交换前: x=" << x << ", y=" << y << std::endl;
    swap(x, y);
    std::cout << "交换后: x=" << x << ", y=" << y << std::endl;

    // 默认参数
    std::cout << "power(3) = " << power(3) << std::endl;       // 9（默认平方）
    std::cout << "power(2,10) = " << power(2, 10) << std::endl; // 1024

    // 函数重载
    std::cout << "max(3,7) = " << maxVal(3, 7) << std::endl;
    std::cout << "max(3.5,2.1) = " << maxVal(3.5, 2.1) << std::endl;
    std::cout << "max(3,7,5) = " << maxVal(3, 7, 5) << std::endl;

    // 递归
    std::cout << "5! = " << factorial(5) << std::endl;  // 120
    std::cout << "fib(10) = " << fib(10) << std::endl;  // 55

    // const 引用传递
    std::string longText = "这是一段很长的文本，使用 const 引用避免拷贝开销";
    printString(longText);

    return 0;
}
```

### 总结

本讲讲解了 C++ 函数的定义与使用。关键要点：第一，值传递创建副本不影响实参，引用传递直接操作原变量，选择哪种方式取决于是否需要修改实参以及性能考量；第二，对于大对象（如 `std::string`、`std::vector`），应使用 `const` 引用传递，既避免拷贝开销又防止意外修改；第三，函数重载让同一操作可以用统一的名字表达，提高了代码可读性，但重载函数的参数列表必须不同（仅返回类型不同不算重载）。常见注意事项：默认参数必须从右向左连续设置，不能跳过中间参数；递归必须有终止条件，否则会无限递归导致栈溢出；朴素递归的斐波那契存在大量重复计算，n 较大时效率极低，应改用迭代或记忆化；函数声明（原型）应放在头文件中，定义放在源文件中，以支持多文件编译。

## 第6讲：数组与字符串

### 概念

数组是存储同类型元素的连续内存块，是 C++ 最基本的复合数据类型。C++ 提供两种数组：**C 风格数组**（`int arr[10];`）和 **C++ 标准库的 `std::array`**（固定大小）与 `std::vector`（动态大小）。字符串方面，C++ 同时支持 C 风格字符串（以 `\0` 结尾的 `char` 数组）和 `std::string` 类。现代 C++ 编程强烈推荐使用 `std::vector` 和 `std::string`，它们比 C 风格版本更安全、更易用，能自动管理内存，避免缓冲区溢出等常见错误。

### 原理

C 风格数组在内存中是连续存储的，数组名实际上是首元素的地址（指针），因此数组不能整体赋值、不能作为函数返回值（栈上数组）。数组的大小在编译期确定（C 风格数组），访问越界不会进行运行时检查，这是 C/C++ 诸多安全漏洞的根源。`std::vector` 则是动态数组，内部维护一块连续内存，当元素数量超过容量时会自动重新分配更大的内存并迁移元素，均摊时间复杂度为 O(1)。`std::string` 类似地封装了字符数组，自动管理内存，提供丰富的成员函数（查找、拼接、子串、比较等）。理解这些容器背后的内存模型，有助于编写高效且安全的代码。

### 例子

```cpp
#include <iostream>
#include <array>
#include <vector>
#include <string>
#include <cstring>

int main() {
    // 1. C 风格数组
    int arr[5] = {10, 20, 30, 40, 50};
    std::cout << "C 数组: ";
    for (int i = 0; i < 5; ++i) {
        std::cout << arr[i] << " ";
    }
    std::cout << std::endl;

    // 2. std::array（固定大小，C++11，更安全）
    std::array<int, 5> arr2 = {1, 2, 3, 4, 5};
    std::cout << "array 大小: " << arr2.size() << std::endl;
    std::cout << "array[2] = " << arr2[2] << std::endl;
    std::cout << "array.at(2) = " << arr2.at(2) << std::endl;  // 带越界检查

    // 3. std::vector（动态数组，最常用）
    std::vector<int> vec = {100, 200, 300};
    vec.push_back(400);   // 尾部添加元素
    vec.push_back(500);
    std::cout << "vector 元素: ";
    for (int v : vec) {
        std::cout << v << " ";
    }
    std::cout << "\nvector 大小: " << vec.size()
              << ", 容量: " << vec.capacity() << std::endl;

    vec.pop_back();       // 删除尾部元素
    std::cout << "pop_back 后大小: " << vec.size() << std::endl;

    // 二维 vector
    std::vector<std::vector<int>> matrix = {
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9}
    };
    std::cout << "矩阵 [1][2] = " << matrix[1][2] << std::endl;  // 6

    // 4. C 风格字符串
    char cstr[] = "Hello";
    std::cout << "C 字符串长度: " << strlen(cstr) << std::endl;  // 5（不含 \0）

    // 5. std::string（推荐使用）
    std::string s1 = "Hello";
    std::string s2 = "World";
    std::string s3 = s1 + " " + s2;   // 拼接
    std::cout << "拼接: " << s3 << std::endl;

    std::cout << "长度: " << s3.length() << std::endl;
    std::cout << "子串: " << s3.substr(0, 5) << std::endl;   // Hello
    std::cout << "查找: " << s3.find("World") << std::endl;  // 返回下标 6

    // 字符串比较
    if (s1 == "Hello") {
        std::cout << "s1 等于 Hello" << std::endl;
    }

    // string 转 C 字符串
    const char* cs = s3.c_str();
    std::cout << "c_str(): " << cs << std::endl;

    // 遍历 string
    for (char c : s1) {
        std::cout << c << "-";
    }
    std::cout << std::endl;

    return 0;
}
```

### 总结

本讲介绍了 C++ 的数组与字符串。关键要点：第一，现代 C++ 应优先使用 `std::vector`（动态数组）和 `std::string`，它们自动管理内存、提供边界检查（`at()` 方法）、接口丰富，远比 C 风格版本安全；第二，`std::array` 是 C 风格数组的现代替代品，固定大小但支持 STL 接口，适合大小已知的场景；第三，`std::string` 支持拼接、查找、子串、比较等丰富操作，且与 C 风格字符串可通过 `c_str()` 互转。常见注意事项：C 风格数组不进行越界检查，访问越界是未定义行为，可能导致程序崩溃或安全漏洞；`vector` 的 `[]` 运算符也不检查越界，需要安全检查时使用 `at()`；`vector` 在大量 `push_back` 时可能频繁扩容，若已知大小应使用 `reserve()` 预分配容量以提升性能；C 风格字符串以 `\0` 结尾，操作时需注意缓冲区大小，避免 `strcpy` 等函数的溢出风险。

## 第7讲：指针与引用

### 概念

指针和引用是 C++ 中两个密切相关的概念，也是 C++ 区别于许多高级语言的核心特性。**指针**是一个变量，其值为另一个变量的内存地址；通过指针可以间接访问和修改它所指向的数据。**引用**则是某个变量的别名，声明后必须立即初始化且不可重新绑定到其他变量。指针更灵活但更危险（可能为空、可能悬空、需要手动管理），引用更安全但功能受限（必须初始化、不可重绑、没有"空引用"）。理解指针和引用的区别与联系，是掌握 C++ 内存模型、函数参数传递、动态内存管理以及后续面向对象特性的基础。

### 原理

指针的本质是存储内存地址的变量。在 64 位系统上，一个指针占 8 字节，无论它指向什么类型——类型信息只决定了解引用时如何解释那块内存。指针的典型操作包括：取地址（`&`）、解引用（`*`）、指针算术（`+`、`-`，按指向类型的大小步进）。引用在底层通常由编译器实现为常量指针，但对程序员而言它表现为别名——一旦绑定就不可改变，使用时无需解引用，语法更自然。指针的危险性在于：未初始化的指针包含随机地址，解引用会导致未定义行为；`delete` 后的指针成为悬空指针，再次使用会引发崩溃；指针算术越界同样危险。引用则规避了这些问题——它必须初始化、不可为空、不可重绑，因此现代 C++ 编程中，能用引用的地方就应优先使用引用，仅在需要"可空"或"可重绑"语义时才使用指针。

### 例子

```cpp
#include <iostream>

int main() {
    // 1. 指针基础
    int value = 42;
    int* ptr = &value;   // 取地址赋给指针
    std::cout << "value = " << value << std::endl;
    std::cout << "ptr 指向的地址: " << ptr << std::endl;
    std::cout << "*ptr (解引用) = " << *ptr << std::endl;

    *ptr = 100;          // 通过指针修改原变量
    std::cout << "修改后 value = " << value << std::endl;  // 100

    // 2. 引用基础
    int a = 10;
    int& ref = a;        // ref 是 a 的别名
    ref = 20;            // 修改 ref 即修改 a
    std::cout << "a = " << a << std::endl;  // 20

    // int& badRef;       // 编译错误：引用必须初始化
    // ref = b;           // 这是赋值，不是重绑定——会改变 a 的值

    // 3. 指针与引用在函数中的对比
    // （此处用代码演示，函数定义见下方说明）

    // 4. 空指针
    int* nullPtr = nullptr;  // C++11 推荐的空指针写法
    if (nullPtr == nullptr) {
        std::cout << "指针为空" << std::endl;
    }
    // *nullPtr;  // 危险！解引用空指针是未定义行为

    // 5. 指针算术（常用于数组）
    int arr[] = {10, 20, 30, 40, 50};
    int* p = arr;        // 数组名退化为首元素指针
    std::cout << "*p = " << *p << std::endl;          // 10
    std::cout << "*(p+2) = " << *(p + 2) << std::endl; // 30
    std::cout << "p[3] = " << p[3] << std::endl;       // 40

    // 6. 常量指针与指针常量
    int x = 5, y = 10;
    const int* ptrToConst = &x;   // 指向常量的指针：不能通过指针改值，可改指向
    // *ptrToConst = 6;           // 编译错误
    ptrToConst = &y;              // OK

    int* const constPtr = &x;     // 常量指针：指针本身不可变，可改值
    *constPtr = 6;                // OK
    // constPtr = &y;             // 编译错误

    const int* const both = &x;   // 都不可变

    // 7. 动态内存分配（new / delete）
    int* dynamic = new int(99);   // 堆上分配
    std::cout << "*dynamic = " << *dynamic << std::endl;
    delete dynamic;               // 必须手动释放，否则内存泄漏
    dynamic = nullptr;            // 良好习惯：释放后置空

    // 动态数组
    int* dynArr = new int[5]{1, 2, 3, 4, 5};
    std::cout << "dynArr[2] = " << dynArr[2] << std::endl;
    delete[] dynArr;              // 数组用 delete[]
    dynArr = nullptr;

    return 0;
}
```

**指针 vs 引用对比表**：

| 特性 | 指针 | 引用 |
|------|------|------|
| 是否必须初始化 | 否（但建议） | 是 |
| 是否可为空 | 可以（`nullptr`） | 不可以 |
| 是否可重绑 | 可以 | 不可以 |
| 是否需解引用 | 需要 `*` | 不需要 |
| 语法复杂度 | 较高 | 低 |
| 适用场景 | 动态内存、可空、数组 | 函数参数、别名 |

### 总结

本讲讲解了 C++ 的指针与引用。关键要点：第一，指针存储地址，引用是别名，二者底层相似但语义不同——引用更安全（不可空、不可重绑），指针更灵活（可空、可重绑、支持算术）；第二，现代 C++ 优先使用引用，仅在需要"可空"或"可重绑"时使用指针，且指针应优先使用智能指针（后续章节）而非裸指针；第三，动态内存分配 `new`/`delete` 必须配对使用，数组用 `new[]`/`delete[]`，否则会导致内存泄漏或堆损坏。常见注意事项：解引用空指针或悬空指针是未定义行为，可能崩溃也可能"看似正常"运行，极难排查；`const` 修饰指针时位置很关键——`const int*`（指向常量）、`int* const`（指针常量）、`const int* const`（都常量）含义不同；释放内存后应立即将指针置为 `nullptr`，避免悬空指针；不要返回局部变量的指针或引用，因为局部变量在函数返回后已被销毁。

# 第3章 面向对象编程

面向对象编程（OOP）是 C++ 区别于 C 语言的最重要特性。本章将带你理解类与对象、构造与析构、继承与多态这四大支柱，这是 C++ 程序设计的核心思想。掌握本章，你就能设计出结构良好、易于扩展、可维护性强的软件系统。

---

## 第8讲：类与对象

### 概念

类是 C++ 实现面向对象编程的基本工具，它是一种用户自定义的数据类型，将**数据（成员变量）**和**操作数据的方法（成员函数）**封装在一起。对象则是类的实例——类是蓝图，对象是根据蓝图创建的具体实体。类的成员有三种访问权限：`public`（公有，外部可访问）、`private`（私有，仅类内可访问）、`protected`（保护，类内及派生类可访问）。封装是面向对象的第一大特性，通过将数据设为私有、提供公有方法来访问，可以隐藏内部实现细节，保护数据完整性，这就是"数据隐藏"原则。

### 原理

类的封装机制基于**访问控制**。编译器在编译期检查所有对类成员的访问，违反访问权限的代码会编译失败。将成员变量设为 `private`、提供 `public` 的 getter/setter 方法，可以确保所有对数据的访问都经过验证——例如设置年龄时检查是否为负数。这种设计使得类的内部实现可以自由变化（如改变存储方式），而不影响使用该类的代码，这就是封装带来的"实现与接口分离"优势。类定义本身不占用内存（它只是类型描述），只有创建对象时才分配内存。对象的内存布局通常是成员变量连续排列（可能有对齐填充），成员函数则存储在代码段，所有对象共享同一份函数代码，通过隐式的 `this` 指针区分不同对象。`this` 指针指向调用该函数的对象本身，是理解成员函数机制的关键。

### 例子

```cpp
#include <iostream>
#include <string>

// 定义一个 Student 类
class Student {
private:                          // 私有成员：数据隐藏
    std::string name;
    int age;
    double score;

public:                           // 公有成员：对外接口
    // 构造函数（下讲详解）
    Student(std::string n, int a, double s) : name(n), age(a), score(s) {}

    // Getter 方法
    std::string getName() const { return name; }
    int getAge() const { return age; }
    double getScore() const { return score; }

    // Setter 方法（带数据验证）
    void setAge(int a) {
        if (a < 0 || a > 150) {
            std::cout << "年龄不合法!" << std::endl;
            return;
        }
        age = a;
    }

    void setScore(double s) {
        if (s < 0 || s > 100) {
            std::cout << "分数不合法!" << std::endl;
            return;
        }
        score = s;
    }

    // 成员函数：显示信息
    void display() const {
        std::cout << "姓名: " << name
                  << ", 年龄: " << age
                  << ", 分数: " << score << std::endl;
    }

    // 成员函数：判断是否优秀
    bool isExcellent() const {
        return score >= 90;
    }
};

int main() {
    // 创建对象
    Student s1("张三", 20, 95.5);
    Student s2("李四", 22, 78.0);

    // 调用成员函数
    s1.display();
    s2.display();

    // 通过 getter 访问私有数据
    std::cout << s1.getName() << " 的分数: " << s1.getScore() << std::endl;

    // 通过 setter 修改数据（带验证）
    s2.setScore(85.0);
    s2.setAge(-5);    // 非法值，会被拒绝
    s2.display();

    // 判断是否优秀
    if (s1.isExcellent()) {
        std::cout << s1.getName() << " 是优秀学生" << std::endl;
    }

    // 对象指针
    Student* ptr = &s1;
    ptr->display();              // 通过指针访问成员用 ->
    std::cout << ptr->getAge() << std::endl;

    return 0;
}
```

### 总结

本讲讲解了 C++ 类与对象的基础。关键要点：第一，类是数据与操作的封装，对象是类的实例，通过访问控制（`public`/`private`/`protected`）实现数据隐藏，这是封装的核心；第二，成员变量应设为 `private`，通过 `public` 的 getter/setter 提供受控访问，这样可以在 setter 中加入数据验证，保护对象状态的有效性；第三，`this` 指针是成员函数的隐式参数，指向调用对象，通过它成员函数才能操作正确的对象数据。常见注意事项：类的声明末尾必须有分号，这是初学者常犯的语法错误；不修改对象状态的成员函数应声明为 `const`（如 `getName() const`），这样 `const` 对象也能调用；通过对象访问成员用 `.`，通过指针访问用 `->`；类内默认访问权限是 `private`，`struct` 默认是 `public`，除此之外二者几乎等价。

## 第9讲：构造函数与析构函数

### 概念

构造函数和析构函数是类中两个特殊的成员函数，负责对象的**初始化**和**清理**。构造函数在对象创建时自动调用，与类同名、无返回类型，用于初始化成员变量。构造函数有多种形式：**默认构造函数**（无参）、**带参构造函数**、**拷贝构造函数**（用同类型对象初始化）、**移动构造函数**（C++11，窃取资源）。析构函数在对象销毁时自动调用，名为 `~类名`，无参无返回，用于释放对象持有的资源（如动态内存、文件句柄）。正确编写构造与析构函数，是管理对象生命周期的关键，也是 RAII（资源获取即初始化）思想的基础。

### 原理

构造函数的调用时机由对象的创建方式决定：栈上局部对象在声明处构造、函数返回时析构；堆上对象在 `new` 时构造、`delete` 时析构；全局/静态对象在 `main` 之前构造、程序结束时析构。**成员初始化列表**（`:` 后的部分）在构造函数体执行之前初始化成员，效率高于在函数体内赋值——尤其对于 `const` 成员、引用成员和没有默认构造函数的类类型成员，必须使用初始化列表。拷贝构造函数在三种情况下被调用：用同类型对象初始化新对象、按值传递函数参数、按值返回函数结果。C++11 引入**移动语义**后，移动构造函数通过"窃取"临时对象（右值）的资源而非拷贝，大幅提升了性能——例如 `std::vector` 的移动只需交换几个指针，而拷贝需要复制所有元素。析构函数的调用顺序与构造相反：后构造的先析构（栈的后进先出特性）。

### 例子

```cpp
#include <iostream>
#include <string>

class StringWrapper {
private:
    char* data;           // 动态分配的字符数组
    size_t length;

public:
    // 1. 默认构造函数
    StringWrapper() : data(nullptr), length(0) {
        std::cout << "  [默认构造] 空对象" << std::endl;
    }

    // 2. 带参构造函数（使用初始化列表）
    StringWrapper(const char* str) : length(0) {
        if (str) {
            while (str[length] != '\0') ++length;
            data = new char[length + 1];
            for (size_t i = 0; i <= length; ++i) data[i] = str[i];
        } else {
            data = nullptr;
        }
        std::cout << "  [带参构造] \"" << (data ? data : "") << "\"" << std::endl;
    }

    // 3. 拷贝构造函数（深拷贝）
    StringWrapper(const StringWrapper& other) : length(other.length) {
        if (other.data) {
            data = new char[length + 1];
            for (size_t i = 0; i <= length; ++i) data[i] = other.data[i];
        } else {
            data = nullptr;
        }
        std::cout << "  [拷贝构造] 深拷贝 \"" << (data ? data : "") << "\"" << std::endl;
    }

    // 4. 移动构造函数（C++11，窃取资源）
    StringWrapper(StringWrapper&& other) noexcept
        : data(other.data), length(other.length) {
        other.data = nullptr;     // 置空源对象，防止双重释放
        other.length = 0;
        std::cout << "  [移动构造] 窃取资源" << std::endl;
    }

    // 5. 析构函数
    ~StringWrapper() {
        std::cout << "  [析构] 释放 \"" << (data ? data : "") << "\"" << std::endl;
        delete[] data;            // 释放动态内存
    }

    void print() const {
        std::cout << "内容: \"" << (data ? data : "") << "\"" << std::endl;
    }
};

// 函数按值传递和返回，触发拷贝/移动
StringWrapper makeString(const char* s) {
    StringWrapper temp(s);
    return temp;  // 可能触发移动构造（或 RVO 优化）
}

int main() {
    std::cout << "=== 创建对象 ===" << std::endl;
    StringWrapper s1;                  // 默认构造
    StringWrapper s2("Hello");         // 带参构造

    std::cout << "\n=== 拷贝构造 ===" << std::endl;
    StringWrapper s3 = s2;             // 拷贝构造
    s3.print();

    std::cout << "\n=== 移动构造 ===" << std::endl;
    StringWrapper s4 = std::move(s2);  // 移动构造，s2 变为空
    s4.print();
    std::cout << "s2 移动后: ";
    s2.print();                        // s2 的 data 已被窃取

    std::cout << "\n=== 函数返回 ===" << std::endl;
    StringWrapper s5 = makeString("World");
    s5.print();

    std::cout << "\n=== 离开作用域，析构顺序 ===" << std::endl;
    return 0;
}
```

### 总结

本讲讲解了构造函数与析构函数。关键要点：第一，构造函数负责初始化、析构函数负责清理，二者自动调用，是对象生命周期管理的基石；第二，应优先使用**成员初始化列表**而非函数体内赋值，前者更高效且对某些成员（`const`、引用、无默认构造的类型）是必须的；第三，当类持有动态资源（如 `new` 分配的内存）时，必须自定义**拷贝构造函数**实现深拷贝，否则默认的浅拷贝会导致双重释放；第四，移动构造函数通过窃取临时对象资源大幅提升性能，是现代 C++ 的重要优化手段。常见注意事项：遵循"三/五法则"——如果需要自定义析构函数、拷贝构造或拷贝赋值之一，通常三者都需要（C++11 后加上移动构造和移动赋值共五个）；析构函数应声明为 `virtual`（下讲继承中详解）以确保通过基类指针删除派生类对象时正确析构；移动构造函数应加 `noexcept` 修饰，否则某些容器（如 `vector`）在扩容时会退化为拷贝以保证异常安全。

## 第10讲：继承

### 概念

继承是面向对象的第二大特性，它允许创建一个新类（**派生类**/子类）从已有类（**基类**/父类）继承数据和操作，从而实现代码复用和层次化设计。C++ 支持三种继承方式：`public`（最常用，基类的 `public`/`protected` 成员在派生类中保持原访问权限）、`protected`、`private`（默认）。继承还分为**单继承**（一个直接基类）和**多继承**（多个直接基类）。通过继承，可以建立"is-a"（是一个）关系——例如"学生是人"、"汽车是交通工具"。继承配合虚函数实现多态，是面向对象设计的核心。

### 原理

继承的内存模型是派生类对象包含一个基类子对象加上派生类自己的成员。构造派生类对象时，先调用基类构造函数初始化基类部分，再初始化派生类部分；析构时顺序相反，先析构派生类部分再析构基类部分。访问权限在继承中的变化：`public` 继承保持基类成员的访问属性，`protected` 继承将基类 `public` 成员变为 `protected`，`private` 继承将基类所有可访问成员变为 `private`。多继承会带来**菱形继承**问题——若 B 和 C 都继承自 A，D 同时继承 B 和 C，则 D 中包含两份 A 的成员，造成二义性和内存浪费。C++ 通过**虚继承**（`virtual` 继承）解决此问题，使共同基类只保留一份实例。继承应当表达 "is-a" 关系，而非 "has-a"（拥有）关系——后者应使用组合（成员变量）而非继承。

### 例子

```cpp
#include <iostream>
#include <string>

// 基类：Person
class Person {
protected:                    // 派生类可访问
    std::string name;
    int age;

public:
    Person(const std::string& n, int a) : name(n), age(a) {
        std::cout << "  Person 构造" << std::endl;
    }

    virtual ~Person() {       // 虚析构（下讲详解）
        std::cout << "  Person 析构" << std::endl;
    }

    virtual void introduce() const {   // 虚函数，可被重写
        std::cout << "我是 " << name << "，今年 " << age << " 岁" << std::endl;
    }
};

// 派生类：Student（public 继承 Person）
class Student : public Person {
private:
    std::string school;

public:
    // 调用基类构造函数初始化基类部分
    Student(const std::string& n, int a, const std::string& s)
        : Person(n, a), school(s) {
        std::cout << "  Student 构造" << std::endl;
    }

    ~Student() override {
        std::cout << "  Student 析构" << std::endl;
    }

    // 重写基类虚函数
    void introduce() const override {
        std::cout << "我是 " << name << "，" << age << " 岁，就读于 "
                  << school << std::endl;
    }

    void study() const {
        std::cout << name << " 正在学习" << std::endl;
    }
};

// 派生类：Teacher
class Teacher : public Person {
private:
    std::string subject;

public:
    Teacher(const std::string& n, int a, const std::string& sub)
        : Person(n, a), subject(sub) {}

    void introduce() const override {
        std::cout << "我是 " << name << " 老师，教授 " << subject << std::endl;
    }

    void teach() const {
        std::cout << name << " 正在讲授 " << subject << std::endl;
    }
};

int main() {
    std::cout << "=== 创建 Student ===" << std::endl;
    Student stu("张三", 20, "清华大学");
    stu.introduce();    // 调用派生类版本
    stu.study();

    std::cout << "\n=== 创建 Teacher ===" << std::endl;
    Teacher tea("李老师", 45, "数学");
    tea.introduce();
    tea.teach();

    std::cout << "\n=== 基类指针指向派生类对象 ===" << std::endl;
    Person* p1 = &stu;
    Person* p2 = &tea;
    p1->introduce();    // 多态：调用 Student::introduce
    p2->introduce();    // 多态：调用 Teacher::introduce

    std::cout << "\n=== 析构顺序 ===" << std::endl;
    return 0;
}
```

### 总结

本讲讲解了 C++ 的继承机制。关键要点：第一，继承表达 "is-a" 关系，派生类自动获得基类的成员，实现代码复用；第二，构造顺序是"基类→派生类"，析构顺序相反，派生类构造函数通过初始化列表调用基类构造函数；第三，`public` 继承最常用，保持基类接口不变，`protected`/`private` 继承较少使用；第四，基类的析构函数应声明为 `virtual`，确保通过基类指针删除派生类对象时调用正确的析构函数，避免资源泄漏。常见注意事项：不要滥用继承——如果关系是 "has-a"（如汽车有引擎），应使用组合而非继承；多继承要警惕菱形问题，必要时使用虚继承；`override` 关键字（C++11）能帮助编译器检查是否正确重写了虚函数，强烈建议使用；派生类不能访问基类的 `private` 成员，但可以访问 `protected` 成员，`protected` 为继承而设计。

## 第11讲：多态与虚函数

### 概念

多态是面向对象的第三大特性，指同一接口对不同对象表现出不同行为。C++ 的多态分为**静态多态**（编译期决定，如函数重载、模板）和**动态多态**（运行期决定，通过虚函数实现）。动态多态是本讲重点：通过基类指针或引用调用虚函数时，实际执行的是对象真实类型（派生类）的版本，而非指针/引用声明类型的版本。虚函数通过 `virtual` 关键字声明，派生类可用 `override` 重写。**纯虚函数**（`= 0`）使类成为抽象类，不可实例化，只能作为接口被继承。多态是开闭原则（对扩展开放、对修改关闭）的实现基础，让程序易于扩展新类型而无需修改已有代码。

### 原理

虚函数的底层机制是**虚函数表（vtable）**和**虚表指针（vptr）**。每个含有虚函数的类都有一张虚函数表，存储该类所有虚函数的地址；每个该类的对象都包含一个隐藏的虚表指针，指向所属类的虚表。当通过基类指针调用虚函数时，程序先通过对象的 vptr 找到虚表，再根据函数在虚表中的偏移找到实际要执行的函数地址——这就是"动态绑定"。由于这一查找发生在运行期，所以能根据对象的真实类型调用正确的函数版本。非虚函数则是"静态绑定"，编译期就根据指针/引用的声明类型决定调用哪个版本，效率更高但不具备多态性。纯虚函数在虚表中通常存放为 `nullptr` 或一个抛异常的占位函数，强制派生类必须实现它。虚函数会带来微小性能开销（一次间接寻址）和每个对象多一个指针的内存开销，但带来的设计灵活性远超这些代价。

### 例子

```cpp
#include <iostream>
#include <vector>
#include <memory>

// 抽象基类：Shape（图形）
class Shape {
public:
    virtual ~Shape() = default;

    // 纯虚函数：必须在派生类中实现
    virtual double area() const = 0;
    virtual double perimeter() const = 0;

    // 普通虚函数：有默认实现，派生类可选择重写
    virtual void describe() const {
        std::cout << "这是一个图形，面积=" << area()
                  << "，周长=" << perimeter() << std::endl;
    }
};

// 派生类：Circle（圆形）
class Circle : public Shape {
private:
    double radius;

public:
    Circle(double r) : radius(r) {}

    double area() const override {
        return 3.14159265 * radius * radius;
    }

    double perimeter() const override {
        return 2 * 3.14159265 * radius;
    }

    void describe() const override {
        std::cout << "圆形（半径=" << radius << "）：面积="
                  << area() << "，周长=" << perimeter() << std::endl;
    }
};

// 派生类：Rectangle（矩形）
class Rectangle : public Shape {
private:
    double width, height;

public:
    Rectangle(double w, double h) : width(w), height(h) {}

    double area() const override {
        return width * height;
    }

    double perimeter() const override {
        return 2 * (width + height);
    }

    void describe() const override {
        std::cout << "矩形（" << width << "x" << height << "）：面积="
                  << area() << "，周长=" << perimeter() << std::endl;
    }
};

// 多态的威力：函数只需关心基类接口，无需知道具体类型
void printShapeInfo(const Shape& s) {
    s.describe();   // 运行期根据实际类型调用
}

int main() {
    Circle c(5.0);
    Rectangle r(3.0, 4.0);

    // 1. 通过基类引用调用（多态）
    std::cout << "=== 基类引用多态调用 ===" << std::endl;
    printShapeInfo(c);   // 调用 Circle::describe
    printShapeInfo(r);   // 调用 Rectangle::describe

    // 2. 通过基类指针 + 容器管理多种类型（多态的经典应用）
    std::cout << "\n=== 容器中多态调用 ===" << std::endl;
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(2.0));
    shapes.push_back(std::make_unique<Rectangle>(3.0, 5.0));
    shapes.push_back(std::make_unique<Circle>(10.0));

    double totalArea = 0;
    for (const auto& s : shapes) {
        s->describe();
        totalArea += s->area();
    }
    std::cout << "所有图形总面积: " << totalArea << std::endl;

    // 3. 抽象类不能实例化
    // Shape s;  // 编译错误：不能声明抽象类的对象

    return 0;
}
```

### 总结

本讲讲解了多态与虚函数，这是面向对象设计的精华。关键要点：第一，动态多态通过虚函数实现，基类指针/引用调用虚函数时会根据对象真实类型执行对应版本，这是"同一接口、多种行为"的核心；第二，纯虚函数（`= 0`）定义接口契约，使类成为抽象类不可实例化，强制派生类实现，这是面向接口编程的基础；第三，多态让代码"对扩展开放、对修改关闭"——新增一种图形只需新增一个派生类，无需修改 `printShapeInfo` 等使用基类接口的函数。常见注意事项：基类析构函数必须声明为 `virtual`（或 `= default` 的虚析构），否则通过基类指针 `delete` 派生类对象时只会调用基类析构，导致派生部分资源泄漏；派生类重写虚函数应加 `override` 关键字，让编译器帮你检查签名是否匹配；不要在构造函数和析构函数中调用虚函数——此时对象类型尚未完全建立或已部分销毁，虚函数不会表现出多态行为；多态需要通过指针或引用调用，直接按值传递对象会发生"对象切片"（派生部分被截断），失去多态性。

## 第12讲：模板

### 概念

模板是 C++ 实现泛型编程的机制，它允许你编写"类型无关"的代码——一份代码可以适用于多种数据类型。模板分为**函数模板**和**类模板**两种。函数模板定义一个通用的函数蓝图，编译器根据实际调用时的类型参数自动生成具体函数（实例化）；类模板则定义一个通用的类蓝图，如 `std::vector<T>` 可以是 `int`、`double` 或任意类型的向量。模板是 C++ 标准库（STL）的基石，`vector`、`map`、`sort` 等全部基于模板实现。掌握模板，你就能写出高度复用、类型安全的通用代码。

### 原理

模板的核心机制是**实例化**——编译器在编译期根据模板和具体类型参数生成实际的代码。例如，当你调用 `max<int>(3, 5)` 和 `max<double>(2.1, 3.7)` 时，编译器会生成两个版本的 `max` 函数，分别处理 `int` 和 `double`。这种"代码生成"发生在编译期，因此模板也被称为"编译期多态"或"静态多态"，相比虚函数的运行期多态没有性能开销。模板参数推导让大多数情况下无需显式指定类型（如 `max(3, 5)` 自动推导为 `int`）。模板的强大之处在于它不仅支持类型参数，还支持非类型参数（如整数）和模板模板参数。C++14 引入的**变量模板**、C++17 的 `if constexpr`、C++20 的**概念（Concepts）**进一步增强了模板的表达力和错误信息的可读性。模板的代价是错误信息往往冗长难懂，且会增加编译时间和代码体积（每个类型生成一份代码）。

### 例子

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <stdexcept>

// 1. 函数模板：返回较大值
template <typename T>
T maxValue(const T& a, const T& b) {
    return (a > b) ? a : b;
}

// 2. 函数模板：打印任意类型的容器
template <typename Container>
void printContainer(const Container& c) {
    for (const auto& elem : c) {
        std::cout << elem << " ";
    }
    std::cout << std::endl;
}

// 3. 类模板：一个简单的栈
template <typename T, size_t MaxSize = 100>
class Stack {
private:
    T data[MaxSize];
    size_t top;

public:
    Stack() : top(0) {}

    void push(const T& value) {
        if (top >= MaxSize) {
            throw std::overflow_error("栈已满");
        }
        data[top++] = value;
    }

    T pop() {
        if (top == 0) {
            throw std::underflow_error("栈为空");
        }
        return data[--top];
    }

    bool empty() const { return top == 0; }
    size_t size() const { return top; }
};

// 4. 模板特化：为特定类型提供特殊实现
template <typename T>
class TypeInfo {
public:
    static void print() { std::cout << "未知类型" << std::endl; }
};

// 对 int 的特化
template <>
class TypeInfo<int> {
public:
    static void print() { std::cout << "这是 int 类型" << std::endl; }
};

// 5. 非类型模板参数
template <int N>
struct Factorial {
    static const int value = N * Factorial<N - 1>::value;  // 编译期递归
};
template <>
struct Factorial<0> {
    static const int value = 1;   // 递归终止
};

int main() {
    // 函数模板
    std::cout << "max(3, 7) = " << maxValue(3, 7) << std::endl;
    std::cout << "max(2.5, 1.8) = " << maxValue(2.5, 1.8) << std::endl;
    std::cout << "max('a','z') = " << maxValue('a', 'z') << std::endl;
    std::cout << "max(string) = " << maxValue(std::string("apple"),
                                              std::string("banana")) << std::endl;

    // 类模板
    Stack<int> intStack;
    intStack.push(10);
    intStack.push(20);
    intStack.push(30);
    std::cout << "栈大小: " << intStack.size() << std::endl;
    std::cout << "出栈: " << intStack.pop() << std::endl;

    Stack<std::string, 50> strStack;   // 指定非类型参数
    strStack.push("Hello");
    strStack.push("World");
    std::cout << "出栈: " << strStack.pop() << std::endl;

    // 模板特化
    TypeInfo<int>::print();
    TypeInfo<double>::print();

    // 编译期计算
    std::cout << "5! = " << Factorial<5>::value << std::endl;  // 120

    return 0;
}
```

### 总结

本讲讲解了 C++ 模板。关键要点：第一，模板是编译期代码生成机制，实现类型无关的泛型编程，是 STL 的基石；第二，函数模板和类模板是最常用的两种形式，编译器根据类型参数自动实例化具体代码；第三，模板支持特化（为特定类型提供特殊实现）和非类型参数，还能通过模板递归实现编译期计算（模板元编程）。常见注意事项：模板定义通常放在头文件中，因为编译器需要完整定义才能实例化；模板的错误信息往往非常冗长，初学者应学会定位关键信息（通常在错误信息末尾）；模板会按需实例化，每种类型生成一份代码，可能导致代码体积膨胀（"模板膨胀"）；不要过度使用复杂的模板元编程技巧，那会严重损害代码可读性和可维护性；C++20 的 Concepts 可以约束模板参数，大幅改善错误信息，建议在新代码中使用。

## 第13讲：标准模板库（STL）

### 概念

标准模板库（Standard Template Library, STL）是 C++ 标准库的核心组成部分，提供了一组通用的、高效的、模板化的数据结构和算法。STL 由六大组件构成：**容器**（containers，存储数据，如 `vector`、`list`、`map`、`set`）、**算法**（algorithms，操作数据，如 `sort`、`find`、`copy`）、**迭代器**（iterators，连接容器与算法的"指针"）、**函数对象**（functors，重载 `operator()` 的类）、**适配器**（adapters，如 `stack`、`queue`）和**分配器**（allocators，内存管理）。STL 的设计哲学是"泛型编程"——算法通过迭代器操作容器，与具体容器类型解耦，一份算法可适用于多种容器。

### 原理

STL 的精髓在于**迭代器**作为容器与算法之间的桥梁。迭代器是一种行为类似指针的对象，提供对容器元素的访问能力。不同容器提供不同类别的迭代器：`vector` 和 `array` 提供随机访问迭代器（支持 `+n`、`<` 等），`list` 提供双向迭代器（支持 `++`、`--`），`forward_list` 提供单向迭代器。算法（如 `std::sort`）只依赖迭代器接口，不关心容器具体类型——这就是为什么 `std::sort` 既能排序 `vector` 也能排序普通数组。容器按底层结构分为：**序列容器**（`vector`、`deque`、`list`、`array`，按位置存储）、**关联容器**（`map`、`set`、`multimap`、`multiset`，基于红黑树，有序，O(log n) 查找）、**无序关联容器**（`unordered_map`、`unordered_set`，基于哈希表，平均 O(1) 查找）和**容器适配器**（`stack`、`queue`、`priority_queue`）。理解每种容器的底层结构和时间复杂度，是高效使用 STL 的关键。

### 例子

```cpp
#include <iostream>
#include <vector>
#include <list>
#include <map>
#include <unordered_map>
#include <set>
#include <algorithm>
#include <numeric>
#include <string>

int main() {
    // 1. vector + 算法
    std::vector<int> nums = {5, 2, 8, 1, 9, 3, 7, 4, 6};
    std::sort(nums.begin(), nums.end());           // 排序
    std::cout << "排序后: ";
    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;

    auto it = std::find(nums.begin(), nums.end(), 7);  // 查找
    if (it != nums.end()) std::cout << "找到 7，位置: "
                                     << (it - nums.begin()) << std::endl;

    int sum = std::accumulate(nums.begin(), nums.end(), 0);  // 求和
    std::cout << "总和: " << sum << std::endl;

    auto [minIt, maxIt] = std::minmax_element(nums.begin(), nums.end());
    std::cout << "最小: " << *minIt << ", 最大: " << *maxIt << std::endl;

    // 2. list（双向链表，适合频繁插入删除）
    std::list<std::string> names = {"Alice", "Bob", "Charlie"};
    names.push_front("Zero");
    names.push_back("David");
    names.sort();   // list 有自己的 sort 成员函数
    std::cout << "list: ";
    for (const auto& n : names) std::cout << n << " ";
    std::cout << std::endl;

    // 3. map（有序键值对，红黑树）
    std::map<std::string, int> scores;
    scores["Alice"] = 95;
    scores["Bob"] = 87;
    scores["Charlie"] = 92;
    scores.emplace("David", 88);

    std::cout << "map（有序）:" << std::endl;
    for (const auto& [name, score] : scores) {   // C++17 结构化绑定
        std::cout << "  " << name << ": " << score << std::endl;
    }
    std::cout << "Alice 的分数: " << scores["Alice"] << std::endl;
    std::cout << "Eve 存在吗? " << (scores.count("Eve") ? "是" : "否") << std::endl;

    // 4. unordered_map（哈希表，查找更快但无序）
    std::unordered_map<int, std::string> idToName;
    idToName[1] = "张三";
    idToName[2] = "李四";
    idToName[3] = "王五";
    std::cout << "ID=2: " << idToName[2] << std::endl;

    // 5. set（有序不重复集合）
    std::set<int> uniqueNums = {3, 1, 4, 1, 5, 9, 2, 6, 5};
    std::cout << "set（自动去重排序）: ";
    for (int n : uniqueNums) std::cout << n << " ";
    std::cout << std::endl;

    // 6. 算法：transform、copy_if 等
    std::vector<int> squares(nums.size());
    std::transform(nums.begin(), nums.end(), squares.begin(),
                   [](int x) { return x * x; });
    std::cout << "平方: ";
    for (int n : squares) std::cout << n << " ";
    std::cout << std::endl;

    std::vector<int> evens;
    std::copy_if(nums.begin(), nums.end(), std::back_inserter(evens),
                 [](int x) { return x % 2 == 0; });
    std::cout << "偶数: ";
    for (int n : evens) std::cout << n << " ";
    std::cout << std::endl;

    return 0;
}
```

### 总结

本讲讲解了 STL 这一 C++ 最强大的武器库。关键要点：第一，STL 由容器、算法、迭代器等组件构成，迭代器是连接容器与算法的桥梁，使算法与容器解耦；第二，选择容器要考虑操作特点——频繁随机访问用 `vector`，频繁头尾插入用 `deque`，频繁中间插入删除用 `list`，键值查找用 `map`/`unordered_map`；第三，`unordered_*` 基于哈希表平均 O(1) 查找但无序，`map`/`set` 基于红黑树 O(log n) 查找且有序，按需选择。常见注意事项：`vector` 在尾部操作高效，中间插入删除需移动元素 O(n)；遍历容器时不要修改容器结构（如 `push_back` 可能导致迭代器失效）；`map` 的 `operator[]` 会在键不存在时插入默认值，若只想查找应使用 `find` 或 `at`；优先使用 STL 算法而非手写循环，它们经过优化且更不易出错；`emplace` 比 `push_back` + 临时对象更高效，避免不必要的拷贝。

## 第14讲：异常处理

### 概念

异常处理是 C++ 处理运行时错误的机制，它将"错误检测"与"错误处理"分离，使代码逻辑更清晰。C++ 异常机制由三部分组成：`throw`（抛出异常）、`try`（检测异常）、`catch`（捕获并处理异常）。当 `throw` 执行时，程序会沿调用栈向上查找匹配的 `catch` 块，这一过程称为**栈展开**（stack unwinding）。C++ 标准库定义了 `std::exception` 异常类层次（如 `runtime_error`、`logic_error`、`bad_alloc` 等），自定义异常应继承自 `std::exception` 并重写 `what()` 方法。异常处理让错误处理代码与正常业务代码分离，提升了程序的可读性和健壮性。

### 原理

异常机制的底层涉及**栈展开**。当 `throw` 抛出异常时，运行时系统会从当前函数开始，逐层向上（向调用者）查找匹配的 `catch` 块。在每一层中，该层函数的局部对象会被正常析构（调用析构函数），然后栈帧被弹出——这就是"栈展开"。这一过程保证了即使发生异常，已构造的对象也能被正确清理，是 RAII 能与异常配合的基础。`catch` 块按声明顺序匹配，匹配规则基于类型（可隐式转换），因此应将更具体的异常类型放在前面，`catch(...)` 放在最后捕获所有异常。异常会带来一定性能开销（尤其在抛出时），但现代 C++ 实现中"不抛出异常的路径"几乎零开销。C++11 引入 `noexcept` 关键字标识函数不抛异常，帮助编译器优化并明确接口契约。异常安全分为四个等级：基本保证（不泄漏资源、不变式不被破坏）、强保证（操作要么成功要么回滚）、不抛出保证（`noexcept`）和无保证——设计时应至少达到基本保证。

### 例子

```cpp
#include <iostream>
#include <stdexcept>
#include <vector>
#include <string>

// 自定义异常类，继承 std::exception
class DivisionByZeroError : public std::runtime_error {
public:
    DivisionByZeroError()
        : std::runtime_error("除零错误：除数不能为零") {}
};

class NegativeNumberError : public std::runtime_error {
public:
    NegativeNumberError(const std::string& msg)
        : std::runtime_error(msg) {}
};

// 可能抛出异常的函数
double safeDivide(int a, int b) {
    if (b == 0) {
        throw DivisionByZeroError();   // 抛出异常
    }
    if (a < 0 || b < 0) {
        throw NegativeNumberError("不支持负数: a=" + std::to_string(a)
                                  + ", b=" + std::to_string(b));
    }
    return static_cast<double>(a) / b;
}

// 演示栈展开：RAII 对象在异常时仍正确析构
class Resource {
    std::string name;
public:
    Resource(const std::string& n) : name(n) {
        std::cout << "  获取资源: " << name << std::endl;
    }
    ~Resource() {
        std::cout << "  释放资源: " << name << std::endl;
    }
};

void riskyFunction() {
    Resource r1("文件句柄");
    Resource r2("数据库连接");
    throw std::runtime_error("模拟中途出错");
    // r2、r1 仍会被正确析构（栈展开）
}

int main() {
    // 1. 基本异常处理
    try {
        std::cout << "10 / 2 = " << safeDivide(10, 2) << std::endl;
        std::cout << "10 / 0 = " << safeDivide(10, 0) << std::endl;  // 抛异常
    } catch (const DivisionByZeroError& e) {
        std::cout << "捕获: " << e.what() << std::endl;
    } catch (const NegativeNumberError& e) {
        std::cout << "捕获: " << e.what() << std::endl;
    }

    // 2. 捕获基类异常（多态）
    try {
        safeDivide(-5, 3);
    } catch (const std::exception& e) {   // 捕获所有 std::exception 派生类
        std::cout << "捕获标准异常: " << e.what() << std::endl;
    }

    // 3. 栈展开演示
    std::cout << "\n=== 栈展开演示 ===" << std::endl;
    try {
        riskyFunction();
    } catch (const std::exception& e) {
        std::cout << "主函数捕获: " << e.what() << std::endl;
    }

    // 4. 捕获所有异常
    try {
        throw 42;   // 抛出 int（不推荐，但语法允许）
    } catch (...) {
        std::cout << "捕获了未知类型的异常" << std::endl;
    }

    // 5. 异常重新抛出
    try {
        try {
            throw std::runtime_error("内层异常");
        } catch (const std::exception& e) {
            std::cout << "内层处理: " << e.what() << "，重新抛出" << std::endl;
            throw;   // 重新抛出当前异常
        }
    } catch (const std::exception& e) {
        std::cout << "外层处理: " << e.what() << std::endl;
    }

    return 0;
}
```

### 总结

本讲讲解了 C++ 异常处理机制。关键要点：第一，异常将错误检测与处理分离，`throw` 抛出、`try` 检测、`catch` 处理，栈展开保证局部对象正确析构，这是 RAII 与异常协作的基础；第二，自定义异常应继承 `std::exception` 并重写 `what()`，便于统一用 `catch(const std::exception&)` 捕获；第三，`catch` 按顺序匹配，具体类型在前、`catch(...)` 在最后，可用 `throw;` 重新抛出当前异常。常见注意事项：不要抛出内置类型（如 `int`），应抛出异常类对象；不要在析构函数中抛出异常，栈展开期间析构函数抛异常会导致程序终止；不要用异常替代正常的控制流，异常应仅用于"异常"情况；`new` 失败默认抛出 `std::bad_alloc` 而非返回 `nullptr`；性能敏感或与 C 交互的代码可用 `noexcept` 标识不抛异常的函数；至少保证代码的"基本异常安全"——不泄漏资源、不破坏不变式。

## 第15讲：内存管理与智能指针

### 概念

内存管理是 C++ 区别于 Java、Python 等带垃圾回收语言的核心特征，也是 C++ 高性能的来源之一。C++ 程序的内存分为**栈**（自动管理，函数局部变量）和**堆**（手动管理，`new`/`delete`）。手动管理堆内存容易出错：忘记 `delete` 导致内存泄漏、重复 `delete` 导致崩溃、`delete` 后继续使用导致未定义行为。C++11 引入的**智能指针**解决了这一难题：`std::unique_ptr`（独占所有权）、`std::shared_ptr`（共享所有权，引用计数）、`std::weak_ptr`（弱引用，不增加引用计数，解决循环引用）。智能指针利用 RAII 思想，在析构时自动释放内存，几乎可以完全替代裸指针的 `new`/`delete`，是现代 C++ 内存管理的首选方案。

### 原理

智能指针的核心是 **RAII（Resource Acquisition Is Initialization，资源获取即初始化）** 思想——将资源的生命周期绑定到对象的生命周期，对象析构时自动释放资源。`unique_ptr` 通过**删除拷贝构造、只允许移动**来保证独占所有权，开销几乎为零（与裸指针相当），是默认首选。`shared_ptr` 内部维护一个**引用计数**：每次拷贝 `shared_ptr` 计数加一，每次析构减一，归零时释放资源；这带来一定性能开销（原子操作、控制块内存），但实现了多个指针共享同一对象。`weak_ptr` 是 `shared_ptr` 的"观察者"——它指向 `shared_ptr` 管理的对象但不增加引用计数，用于打破循环引用（两个 `shared_ptr` 互相引用会导致内存泄漏，因为计数永远不归零）。智能指针的正确使用能消除绝大多数内存管理错误，同时保持接近裸指针的性能。

### 例子

```cpp
#include <iostream>
#include <memory>
#include <vector>

class Widget {
    std::string name;
public:
    Widget(const std::string& n) : name(n) {
        std::cout << "  Widget 构造: " << name << std::endl;
    }
    ~Widget() {
        std::cout << "  Widget 析构: " << name << std::endl;
    }
    void use() const { std::cout << "  使用 Widget: " << name << std::endl; }
    const std::string& getName() const { return name; }
};

// 1. unique_ptr：独占所有权
void useUniquePtr() {
    std::cout << "=== unique_ptr ===" << std::endl;
    auto p = std::make_unique<Widget>("独占对象");  // 推荐的创建方式
    p->use();
    // std::unique_ptr<Widget> p2 = p;  // 编译错误：不能拷贝
    std::unique_ptr<Widget> p2 = std::move(p);  // 可以移动，p 变为 nullptr
    if (!p) std::cout << "  p 移动后为空" << std::endl;
    p2->use();
}   // p2 离开作用域，自动析构 Widget

// 2. shared_ptr：共享所有权
void useSharedPtr() {
    std::cout << "\n=== shared_ptr ===" << std::endl;
    auto sp1 = std::make_shared<Widget>("共享对象");
    std::cout << "  引用计数: " << sp1.use_count() << std::endl;  // 1

    {
        auto sp2 = sp1;   // 拷贝，计数 +1
        std::cout << "  sp2 作用域内计数: " << sp1.use_count() << std::endl;  // 2
        sp2->use();
    }   // sp2 析构，计数 -1
    std::cout << "  sp2 离开后计数: " << sp1.use_count() << std::endl;  // 1
}   // sp1 析构，计数归零，Widget 释放

// 3. weak_ptr：解决循环引用
struct Node;
struct Node {
    std::string name;
    std::shared_ptr<Node> next;
    std::weak_ptr<Node> prev;   // 用 weak_ptr 避免循环引用
    Node(const std::string& n) : name(n) {
        std::cout << "  Node 构造: " << name << std::endl;
    }
    ~Node() {
        std::cout << "  Node 析构: " << name << std::endl;
    }
};

void useWeakPtr() {
    std::cout << "\n=== weak_ptr 解决循环引用 ===" << std::endl;
    auto n1 = std::make_shared<Node>("节点1");
    auto n2 = std::make_shared<Node>("节点2");
    n1->next = n2;    // n1 -> n2
    n2->prev = n1;    // n2 -> n1（weak_ptr，不增加计数）

    // 使用 weak_ptr 前需 lock() 提升为 shared_ptr
    if (auto locked = n2->prev.lock()) {
        std::cout << "  n2 的前驱: " << locked->name << std::endl;
    }
    std::cout << "  n1 计数: " << n1.use_count() << std::endl;  // 1
    std::cout << "  n2 计数: " << n2.use_count() << std::endl;  // 1
}   // n1、n2 正确析构（若 prev 用 shared_ptr 则计数永远为 2，泄漏）

int main() {
    useUniquePtr();
    useSharedPtr();
    useWeakPtr();

    // 4. 在容器中使用智能指针
    std::cout << "\n=== 容器中的智能指针 ===" << std::endl;
    std::vector<std::unique_ptr<Widget>> widgets;
    widgets.push_back(std::make_unique<Widget>("W1"));
    widgets.push_back(std::make_unique<Widget>("W2"));
    widgets.push_back(std::make_unique<Widget>("W3"));
    for (const auto& w : widgets) {
        w->use();
    }
    // 容器销毁时，所有 Widget 自动释放

    return 0;
}
```

### 总结

本讲讲解了 C++ 内存管理与智能指针，这是现代 C++ 最重要的进步之一。关键要点：第一，智能指针基于 RAII 思想，将资源生命周期绑定到对象生命周期，析构时自动释放，几乎消除内存泄漏和悬空指针问题；第二，`unique_ptr` 独占所有权、零开销，是默认首选；`shared_ptr` 共享所有权、引用计数，适合所有权不明确或需共享的场景；`weak_ptr` 不增加计数，用于打破循环引用和观察者模式；第三，优先使用 `std::make_unique` 和 `std::make_shared` 创建智能指针，它们更安全（避免内存泄漏窗口）且更高效（`make_shared` 一次分配对象和控制块）。常见注意事项：不要用 `get()` 返回的裸指针初始化另一个智能指针或 `delete` 它；不要用同一个裸指针创建多个 `shared_ptr`，会导致双重释放；`shared_ptr` 的引用计数是原子操作，线程安全，但指向的对象本身不是；现代 C++ 中应尽量避免使用裸 `new`/`delete`，让智能指针成为内存管理的主角。

## 第16讲：Lambda 表达式与函数对象

### 概念

Lambda 表达式是 C++11 引入的**匿名函数**机制，允许在代码中就地定义函数，无需单独声明。Lambda 的语法为 `[捕获列表](参数列表) -> 返回类型 { 函数体 }`。捕获列表指定如何使用外部变量（值捕获 `[=]`、引用捕获 `[&]`、指定变量 `[x]`/`[&x]`）。Lambda 本质上会被编译器转换为一个**闭包类**（closure class）的临时对象，称为闭包。**函数对象**（functor）则是重载了 `operator()` 的类，行为类似函数，是 Lambda 出现之前实现"可调用对象"的传统方式。Lambda 和函数对象广泛用于 STL 算法的自定义比较、过滤、转换等场景，是现代 C++ 函数式编程风格的核心工具。

### 原理

Lambda 的底层实现是一个编译器自动生成的闭包类。例如 `[x](int a) { return a + x; }` 会被转换为类似这样的类：`struct __lambda { int x; int operator()(int a) const { return a + x; } };`，然后创建该类的临时对象。捕获列表对应闭包类的成员变量——值捕获将外部变量拷贝为成员，引用捕获将引用存为成员（需注意被引用对象的生命周期）。Lambda 默认的 `operator()` 是 `const` 的，若需修改捕获的值，需加 `mutable` 关键字。Lambda 的返回类型通常可自动推导（C++11 需单返回语句，C++14 起任意函数体均可推导）。Lambda 可转换为函数指针（仅当无捕获时），因此能传给 C 风格的回调接口。`std::function` 是一个能存储任意可调用对象的类型擦除容器，适合需要存储或传递多种可调用对象的场景，但有运行时开销。`std::bind` 可绑定参数，但在现代 C++ 中大多被 Lambda 取代。

### 例子

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <functional>

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9, 3, 7, 4, 6};

    // 1. 基本 Lambda：作为算法的谓词
    std::sort(nums.begin(), nums.end(), [](int a, int b) {
        return a > b;   // 降序
    });
    std::cout << "降序: ";
    for (int n : nums) std::cout << n << " ";
    std::cout << std::endl;

    // 2. 值捕获 vs 引用捕获
    int threshold = 5;
    int count = 0;

    // 值捕获 [=]：拷贝外部变量（threshold 的副本）
    std::vector<int> greater;
    std::copy_if(nums.begin(), nums.end(), std::back_inserter(greater),
                 [threshold](int x) { return x > threshold; });
    std::cout << "大于 " << threshold << " 的: ";
    for (int n : greater) std::cout << n << " ";
    std::cout << std::endl;

    // 引用捕获 [&]：可修改外部变量
    std::for_each(nums.begin(), nums.end(), [&count](int x) {
        if (x > threshold) ++count;
    });
    std::cout << "大于 " << threshold << " 的个数: " << count << std::endl;

    // 3. mutable Lambda：修改值捕获的副本
    int multiplier = 1;
    auto gen = [multiplier]() mutable {
        return multiplier *= 2;   // 修改的是副本，不影响外部 multiplier
    };
    std::cout << "mutable: " << gen() << " " << gen() << " " << gen() << std::endl;
    std::cout << "外部 multiplier 仍为: " << multiplier << std::endl;  // 1

    // 4. 指定返回类型
    auto divide = [](int a, int b) -> double {
        return static_cast<double>(a) / b;
    };
    std::cout << "7/2 = " << divide(7, 2) << std::endl;

    // 5. 函数对象（functor）：传统可调用对象
    struct Multiplier {
        int factor;
        Multiplier(int f) : factor(f) {}
        int operator()(int x) const { return x * factor; }
    };
    Multiplier triple(3);
    std::cout << "triple(5) = " << triple(5) << std::endl;

    std::vector<int> tripled(nums.size());
    std::transform(nums.begin(), nums.end(), tripled.begin(), Multiplier(3));

    // 6. std::function：存储任意可调用对象
    std::function<int(int, int)> op;
    op = [](int a, int b) { return a + b; };
    std::cout << "加法: " << op(3, 4) << std::endl;
    op = [](int a, int b) { return a * b; };
    std::cout << "乘法: " << op(3, 4) << std::endl;

    // 7. Lambda 递归（需用 std::function）
    std::function<int(int)> fib = [&fib](int n) -> int {
        if (n < 2) return n;
        return fib(n - 1) + fib(n - 2);
    };
    std::cout << "fib(10) = " << fib(10) << std::endl;

    return 0;
}
```

### 总结

本讲讲解了 Lambda 表达式与函数对象。关键要点：第一，Lambda 是就地定义的匿名函数，底层是编译器生成的闭包类，捕获列表决定如何使用外部变量（值捕获拷贝、引用捕获引用）；第二，Lambda 是 STL 算法的最佳搭档，让自定义比较、过滤、转换变得简洁直观，取代了繁琐的函数对象和 `std::bind`；第三，`std::function` 可以存储任意可调用对象（Lambda、函数指针、函数对象），适合需要类型擦除的场景，但有运行时开销。常见注意事项：引用捕获 `[&]` 要警惕悬空引用——若 Lambda 在被捕获变量的作用域外执行（如异步任务），引用可能失效；默认 Lambda 的 `operator()` 是 `const`，需修改值捕获时加 `mutable`；优先使用 Lambda 而非 `std::bind`，前者更清晰高效；性能敏感路径避免用 `std::function`（有类型擦除开销），直接用模板或 Lambda；递归 Lambda 需借助 `std::function` 或 C++14 的泛型 Lambda（`auto` 参数）。

## 第17讲：并发编程基础

### 概念

并发编程是利用多核 CPU 同时执行多个任务的技术，是现代高性能程序的必备能力。C++11 首次在标准库中引入了并发支持，主要包括：`std::thread`（线程）、`std::mutex`（互斥锁）、`std::lock_guard`/`std::unique_lock`（锁的 RAII 包装）、`std::condition_variable`（条件变量）、`std::atomic`（原子操作）、`std::async`/`std::future`（异步任务）等。并发编程的核心挑战是**数据竞争**（data race）——多个线程同时访问共享数据且至少一个在写，会导致未定义行为。通过互斥锁保护共享数据、用原子操作实现无锁同步、用条件变量进行线程间通信，可以编写正确的并发程序。

### 原理

线程是操作系统能调度的最小执行单元，每个线程有自己的栈但共享进程的堆和全局数据。`std::thread` 在创建时启动一个新线程执行指定的函数。当多线程访问共享数据时，若不加以保护，CPU 的乱序执行和缓存一致性会导致数据竞争——例如两个线程同时 `++counter`，可能读到相同的旧值、各自加一后写回，导致只增加了一次而非两次。`std::mutex` 通过操作系统提供的锁机制保证同一时刻只有一个线程进入临界区，`std::lock_guard` 利用 RAII 在构造时加锁、析构时解锁，避免忘记解锁。`std::atomic` 利用 CPU 的原子指令（如 CAS）实现无锁同步，适合简单的计数器等场景，性能优于互斥锁。`std::condition_variable` 允许线程等待某个条件成立，被其他线程通知后继续，是生产者-消费者模式的基础。`std::async` 启动异步任务并返回 `std::future`，可获取结果，是比直接管理线程更高层的抽象。

### 例子

```cpp
#include <iostream>
#include <thread>
#include <mutex>
#include <atomic>
#include <vector>
#include <future>
#include <condition_variable>
#include <queue>

// 1. 基本线程创建
void printMessage(const std::string& msg, int times) {
    for (int i = 0; i < times; ++i) {
        std::cout << msg << " " << i << std::endl;
    }
}

// 2. 数据竞争演示与互斥锁解决
const int THREAD_COUNT = 10;
const int INCREMENTS = 10000;

int unsafeCounter = 0;          // 无保护，会有数据竞争
int safeCounter = 0;            // 用 mutex 保护
std::mutex mtx;
std::atomic<int> atomicCounter(0);  // 原子变量

void incrementUnsafe() {
    for (int i = 0; i < INCREMENTS; ++i) {
        ++unsafeCounter;   // 数据竞争！
    }
}

void incrementSafe() {
    for (int i = 0; i < INCREMENTS; ++i) {
        std::lock_guard<std::mutex> lock(mtx);  // RAII 加锁
        ++safeCounter;
    }
}

void incrementAtomic() {
    for (int i = 0; i < INCREMENTS; ++i) {
        ++atomicCounter;   // 原子操作，无锁
    }
}

// 3. 生产者-消费者（条件变量）
std::queue<int> dataQueue;
std::mutex queueMtx;
std::condition_variable cv;
bool finished = false;

void producer() {
    for (int i = 1; i <= 5; ++i) {
        {
            std::lock_guard<std::mutex> lock(queueMtx);
            dataQueue.push(i);
            std::cout << "生产: " << i << std::endl;
        }
        cv.notify_one();   // 通知消费者
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }
    {
        std::lock_guard<std::mutex> lock(queueMtx);
        finished = true;
    }
    cv.notify_one();
}

void consumer() {
    while (true) {
        std::unique_lock<std::mutex> lock(queueMtx);
        cv.wait(lock, [] { return !dataQueue.empty() || finished; });
        while (!dataQueue.empty()) {
            int val = dataQueue.front();
            dataQueue.pop();
            std::cout << "  消费: " << val << std::endl;
        }
        if (finished) break;
    }
}

int main() {
    // 基本线程
    std::thread t1(printMessage, "线程A", 3);
    std::thread t2(printMessage, "线程B", 3);
    t1.join();   // 等待线程结束
    t2.join();

    // 数据竞争对比
    std::cout << "\n=== 数据竞争演示 ===" << std::endl;
    std::vector<std::thread> threads;

    for (int i = 0; i < THREAD_COUNT; ++i)
        threads.emplace_back(incrementUnsafe);
    for (auto& t : threads) t.join();
    std::cout << "无保护计数器（期望 " << THREAD_COUNT * INCREMENTS
              << "）: " << unsafeCounter << std::endl;

    threads.clear();
    for (int i = 0; i < THREAD_COUNT; ++i)
        threads.emplace_back(incrementSafe);
    for (auto& t : threads) t.join();
    std::cout << "mutex 保护计数器: " << safeCounter << std::endl;

    threads.clear();
    for (int i = 0; i < THREAD_COUNT; ++i)
        threads.emplace_back(incrementAtomic);
    for (auto& t : threads) t.join();
    std::cout << "atomic 计数器: " << atomicCounter << std::endl;

    // 生产者-消费者
    std::cout << "\n=== 生产者-消费者 ===" << std::endl;
    std::thread prod(producer);
    std::thread cons(consumer);
    prod.join();
    cons.join();

    // std::async 异步任务
    std::cout << "\n=== async 异步任务 ===" << std::endl;
    auto future = std::async(std::launch::async, [](int n) {
        long long result = 1;
        for (int i = 2; i <= n; ++i) result *= i;
        return result;
    }, 20);
    std::cout << "20! = " << future.get() << std::endl;  // 阻塞等待结果

    return 0;
}
```

### 总结

本讲讲解了 C++ 并发编程基础。关键要点：第一，多线程共享数据时必须防范数据竞争，用 `std::mutex` 配合 `std::lock_guard`（RAII）保护临界区是最基本手段；第二，`std::atomic` 提供无锁原子操作，适合简单计数器等场景，性能优于互斥锁；第三，`std::condition_variable` 实现线程间等待-通知，是生产者-消费者模式的核心；第四，`std::async`/`std::future` 是比直接管理 `std::thread` 更高层的异步抽象，适合"启动任务、稍后取结果"的场景。常见注意事项：务必调用 `join()` 或 `detach()`，否则线程对象析构会调用 `std::terminate` 终止程序；避免死锁——不要在持有一个锁时去获取另一个锁，或使用 `std::lock` 同时锁定多个锁；条件变量的 `wait` 必须用谓词形式（`cv.wait(lock, predicate)`）以防虚假唤醒；`std::async` 默认策略可能不启动新线程（延迟执行），需显式 `std::launch::async`；并发编程极易出错，调试困难，能用单线程解决就不要引入多线程。

## 第18讲：现代 C++ 新特性（C++11/14/17/20）

### 概念

C++11 是 C++ 历史上最重要的一次升级，被誉为"新 C++"，此后每三年发布一个新标准（C++14、C++17、C++20），持续现代化。本讲梳理各版本的关键特性：**C++11** 引入了 `auto`、`nullptr`、范围 for、Lambda、智能指针、右值引用与移动语义、`constexpr`、强类型枚举等；**C++14** 完善了泛型 Lambda、`auto` 返回类型推导、`std::make_unique`；**C++17** 带来结构化绑定、`if constexpr`、`std::optional`/`std::variant`/`std::any`、文件系统库、并行算法；**C++20** 是又一次大升级，引入 Concepts（概念）、Ranges（范围）、协程、Modules（模块）、`<=>` 飞船运算符等。掌握这些特性，能让你写出更简洁、更安全、更高效的现代 C++ 代码。

### 原理

现代 C++ 的演进遵循"零开销抽象"和"不破坏旧代码"两大原则。`auto` 和类型推导减少了冗余的类型书写，让代码更简洁且不易出错（重构时类型自动跟随）。右值引用与移动语义通过区分"左值"（可取地址、有名字）和"右值"（临时值、将销毁），允许窃取临时对象的资源而非拷贝，大幅提升性能——这是 `std::vector` 等容器返回、`std::string` 拼接等场景的关键优化。`constexpr` 将计算从运行期提前到编译期，既提升运行性能又能用于模板参数。`std::optional` 表达"可能有值也可能没有"，比裸指针或特殊值更安全；`std::variant` 是类型安全的联合体，替代传统 `union`。C++20 的 Concepts 为模板参数添加约束，让模板错误信息从天书变为可读；Ranges 让算法可以链式组合，`views` 实现惰性求值；协程让异步代码可以像同步代码一样书写；Modules 取代头文件包含，大幅加快编译速度。

### 例子

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <optional>
#include <variant>
#include <tuple>
#include <algorithm>
#include <numeric>
// C++20 特性需要较新编译器支持，以下展示可移植的核心特性

// C++14: auto 返回类型推导
template <typename T, typename U>
auto add(T a, U b) {
    return a + b;   // 自动推导返回类型
}

// C++17: if constexpr（编译期分支）
template <typename T>
void process(T value) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << "  整型: " << value << std::endl;
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "  浮点型: " << value << std::endl;
    } else {
        std::cout << "  其他类型" << std::endl;
    }
}

// C++17: std::optional 表示可能失败的操作
std::optional<int> findEven(const std::vector<int>& v) {
    for (int n : v) {
        if (n % 2 == 0) return n;
    }
    return std::nullopt;   // 没找到
}

// C++17: 结构化绑定
std::tuple<std::string, int, double> getPerson() {
    return {"张三", 25, 95.5};
}

int main() {
    // === C++11 特性 ===
    // auto 类型推导
    auto x = 42;              // int
    auto y = 3.14;            // double
    auto s = std::string("hi");

    // nullptr 替代 NULL
    int* p = nullptr;

    // 范围 for
    std::vector<int> v = {1, 2, 3, 4, 5};
    for (const auto& n : v) std::cout << n << " ";
    std::cout << std::endl;

    // 强类型枚举
    enum class Color { Red, Green, Blue };
    Color c = Color::Red;
    // int i = c;  // 编译错误：不能隐式转换

    // === C++14 特性 ===
    std::cout << "add(3, 4.5) = " << add(3, 4.5) << std::endl;  // 7.5

    // 泛型 Lambda（auto 参数）
    auto gl = [](auto a, auto b) { return a + b; };
    std::cout << "gl(1,2)=" << gl(1, 2) << ", gl(1.5,2.5)=" << gl(1.5, 2.5) << std::endl;

    // === C++17 特性 ===
    // 结构化绑定
    auto [name, age, score] = getPerson();
    std::cout << name << ", " << age << " 岁, " << score << " 分" << std::endl;

    // if constexpr
    process(42);
    process(3.14);
    process("hello");

    // std::optional
    std::vector<int> nums = {1, 3, 5, 6, 7};
    if (auto result = findEven(nums)) {
        std::cout << "找到偶数: " << *result << std::endl;
    } else {
        std::cout << "没有偶数" << std::endl;
    }

    // std::variant（类型安全的联合体）
    std::variant<int, double, std::string> var;
    var = 42;
    std::cout << "variant(int): " << std::get<int>(var) << std::endl;
    var = "hello";
    std::cout << "variant(string): " << std::get<std::string>(var) << std::endl;

    // if 语句带初始化（C++17）
    if (auto it = std::find(v.begin(), v.end(), 3); it != v.end()) {
        std::cout << "找到 3，位置: " << (it - v.begin()) << std::endl;
    }

    // === C++20 特性（需编译器支持，-std=c++20）===

    return 0;
}
```

**各标准主要特性速查表**：

| 标准 | 关键特性 |
|------|---------|
| C++11 | `auto`、`nullptr`、范围for、Lambda、智能指针、移动语义、`constexpr`、强枚举、`thread`、`async` |
| C++14 | 泛型Lambda、`auto`返回推导、`std::make_unique`、数字字面分隔符 `1'000` |
| C++17 | 结构化绑定、`if constexpr`、`std::optional`/`variant`/`any`、文件系统库、并行算法、fold表达式 |
| C++20 | Concepts、Ranges、协程、Modules、`<=>`运算符、`std::format`、`consteval` |

### 总结

本讲梳理了现代 C++ 各版本的核心特性。关键要点：第一，C++11 是分水岭，引入的 `auto`、智能指针、Lambda、移动语义等彻底改变了 C++ 的编程风格，现代 C++ 代码应全面采用这些特性；第二，C++17 的结构化绑定、`if constexpr`、`std::optional`/`variant` 让代码更简洁、类型更安全，是日常编程的利器；第三，C++20 的 Concepts、Ranges、协程、Modules 是又一次大飞跃，显著提升了模板的可读性、算法的表达力和编译速度。常见注意事项：使用新特性前确认编译器支持版本（用 `__cplusplus` 宏或 CMake 的 `target_compile_features` 检测）；不要为了用新特性而用——`auto` 会降低可读性的地方（如 `auto x = foo()` 看不出类型）应显式写类型；移动语义只对持有资源的类型有意义，简单类型（`int` 等）移动和拷贝无差别；逐步升级旧代码到现代风格，但优先保证正确性；持续关注新标准，C++23、C++26 还在演进，保持学习是 C++ 程序员的常态。
